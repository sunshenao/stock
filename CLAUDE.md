# A 股短线操作手册

每天按四个时段执行。每个时段开始前先读取 `codex/lessons_learned.md`。

---

## 核心策略：集中趋势跟踪（2026-07 验证，主执行口径）

> 走前向验证 5/07→7/10（约两个月）：**+11.92%，最大回撤 6.34%**，同期沪深300 -1.18%。
> 验证工具：`python scripts/walkforward.py`（读真实 scan.json + 真实收盘价，扣 8bp 成本）。

**选股（按真实动量，不是评分）**
- 从当日合格池按 **5 日动量**从高到低排名（合格池 = 流动性≥1亿、非债券/货币、非 QDII、非 `⚠`/加速见顶、非观察/萌芽/衰弱期）。
- 取**动量最强的 2 只**，可同板块（集中吃主线，不强制分散）；按动量定主次，仓位 **60% / 40%**。
- 每 **3 个交易日**重排一次；持仓跌出重排即换成新的最强 2 只。

**仓位（默认满仓，弱市降档不空仓）**
- 默认 **100% 满仓**（60/40）。
- **弱市降 70%**（42/28）：沪深300 收盘跌破 5 日均线 → 次日降到 70%。
- **崩盘日**：沪深300 单日 ≤ -2% → 次日降到 70%。
- 转强（重回 5 日均线上方）→ 恢复 100%。**任何时候不清仓到 0（除非无动量信号）。**

**止损**
- **15% 移动止损**：个股自持仓期间最高价回撤 > 15% → 清出该仓。

**边界（诚实提示）**
- 这是趋势策略：**趋势月吃大肉（6 月 +10.7%），震荡/下跌月只能靠降档保命**，约 ~6%/月，**做不到旱涝保收 10%/月**。
- 单边熊市里偏多策略必然受伤；靠的是"进攻满仓吃趋势、转弱降 70% 控回撤"。
- 每次调整策略后**必须用 `walkforward.py` 在样本外+样本内两个窗口、扣成本重验**，只认两窗口都不劣化的改动（见 [[feedback-walkforward-test]]）。

## 早盘（开盘前 9:00-9:25）

### 目标：结合最新情况给出今日操作建议

- [ ] 读取 `codex/lessons_learned.md`
- [ ] 读取 `codex/stock/current_positions.md`
- [ ] 用 `news-search` 搜隔夜重大新闻（美股三大指数、纳指/费半、热门中概、外盘、政策、行业）
- [ ] 用 `news-search` 搜持仓方向+候选方向的近期催化、舆情热度
- [ ] 判断市场情绪：恐贪/避险/追涨/恐慌 → 仓位策略微调（恐慌加仓、追涨减仓）
- [ ] 用 `hithink-market-query` 查当前持仓的实时价格、主力资金
- [ ] 运行 `python scripts/etf_analyzer.py --date <最新交易日> --hithink` 获取最新方向排名
- [ ] 输出：
  - 当前持仓状态（每只标的 / 盈亏 / 是否触止损）
  - 今日 ETF 方向建议（TOP 3 方向）
  - 今日操作计划（持有 / 加仓 / 减仓 / 换仓）
  - 关键风险提示（隔夜利空、催化到期、接近止损）
- [ ] 对拟加仓/换仓的标的跑 `stock_checkup.py` 做体检

## 午盘（11:30-13:00）

### 目标：判断持仓是否需要变动

- [ ] 用 `hithink-market-query` 查当前持仓上午表现（涨跌幅、主力资金）
- [ ] 判断：
  - 主力反水（净流出 > 1 亿）→ 立即减仓
  - 跌幅 > 2% 且弱于同方向 ETF → 降为观察仓
  - 涨幅 > 3% 且主力净流入 → 持有不动
  - 横盘且缩量 → 保持观察，不操作
- [ ] 若发现新主线方向（盘中 ETF 涨幅榜出现新面孔且量比 > 2），记录为候补，不急于追
- [ ] 输出：午盘持仓状态 + 是否需要调整

---

## 尾盘（14:30-15:00）

### 目标：最终操作决策

- [ ] 用 `hithink-market-query` 查持仓最新价、主力资金
- [ ] 决策矩阵：
  - 持仓浮盈 > 15% 且 5 日动量减速 → 止盈锁利
  - 持仓浮亏且主力流出 → 减半仓
  - 持仓横盘但主力持续流入 → 持有过夜
  - 新方向全天强势 + 主力确认 → 建仓，单票无上限
- [ ] 输出：每只标的的尾盘操作（持有/减仓/加仓/清仓）+ 理由

## 盘后（收盘后）

### 目标：总结 + 选股 + 纠错

### 4.1 当日总结

- [ ] 对比今日实际涨跌 vs 早盘判断 → 记录偏差
- [ ] 列出今日最大失误（一条）和原因
- [ ] 若发现新教训 → **立即写入 `codex/lessons_learned.md`**（格式：L + 递增编号 + 场景/教训/规则三段式）

### 4.2 完整选股（为明日做准备）

严格按以下步骤执行：

**Step 0: 准备**
- [ ] 读取 `codex/lessons_learned.md`，对照检查
- [ ] 读取 `codex/stock/current_positions.md`
- [ ] 读取 `codex/stock/candidate_pool.md`

**Step 1: ETF 扫描**
```
python scripts/etf_analyzer.py --date YYYY-MM-DD --hithink
```
- [ ] 确认报告已生成，有效行情 > 0
- [ ] 确认 `数据质量` 中旧行情已剔除；只有最新实时交易日允许同花顺实时增强，历史日期不得实时回填
- [ ] 只读取“正式组合候选（唯一执行口径）”，原始强度前三和去相关组合只做诊断参考

**Step 2: 市场环境**
- [ ] 涨跌比 → 仓位：<0.5→冰点40%、≥0.5→**100%满仓**。市场状态标签按涨跌比写，不抄扫描
- [ ] 复核美股映射 + 市场情绪（涨跌比趋势、成交量、涨跌停比）

**Step 3: 催化验证（规则 1 — 核心，不是附加）**
- [ ] **每个候选方向必须搜新闻**：`news-search` + `mcp__cn-financial__search_news` + `announcement-search`
- [ ] 确认实质催化（政策/业绩/产品/订单/行业事件/财报）才纳入候选
- [ ] 排名高无新闻 → 降权或排除；同时搜索持仓反向利空
- [ ] 新闻强度决定仓位：重磅催化+扩散期 → 满仓 60%；有催化已发酵 → 正常；无新闻 → 不买
- [ ] 催化超 1 天 → 重验主力；主力流出 → 利好兑现出货

**Step 4: 标的选择**
- [ ] **先检查持仓**（规则 4）：在 TOP2 → 持有；不在 → 检查是否精英级候选触发换仓
- [ ] 最多选 2 个，可只选 1 个；**两个不能同板块**
- [ ] 非冰点 100% 满仓：选 1 个→60%、选 2 个→60%+40%
- [ ] 对 ETF 方向匹配个股，跑 `stock_checkup.py` + `expert_debate.py`

**Step 5: 仓位计算**
```
python scripts/kelly_sizer.py
```
- [ ] 时间止损检查：持仓 > 8 天未盈利 → 减半
- [ ] 跌停检查：个股前日跌幅 > -9% → 竞价卖出

**Step 6: 输出 selection.md**
- [ ] ETF vs 个股对比表
- [ ] 最终组合表 + 建仓纪律 + 偏差核查
- [ ] 运行 `python scripts/selection_guard.py codex/stock/YYYY-MM-DD/selection.md`，未通过则不得作为正式执行方案

**Step 7: 同步持仓台账**
- [ ] 更新 `current_positions.md`
- [ ] 新个股写入 `candidate_pool.md`

## 执行检查清单（每次盘后选股逐条确认）

- [ ] **新闻催化**（规则 1）：每个候选方向搜新闻验证，有催化才执行，无新闻排除
- [ ] **涨跌比仓位**（规则 2）：<0.5→冰点40%、≥0.5→100%满仓
- [ ] **板块分散**（规则 3）：最多 2 个、不同板块、60%+40%
- [ ] **精英换仓**（规则 4）：评分>70+扩散期 或 >持仓+20 才换，否则不动
- [ ] **崩塌清仓**（规则 5）：跌出TOP30评分降>40% → 当天清
- [ ] **回撤熔断**（规则 6）：全仓DD>8% → 集中1个；DD<3% → 恢复100%
- [ ] **主力确认**（规则 7）：主力流出>1亿不主仓；催化超1天重验
- [ ] **不追高**（规则 8）：单日>7%次日不追；20日>15%等分歧
- [ ] **动量极值**（规则 9）：5日>25%/20日>35%不新开
- [ ] **盘中复核**（规则 10）：10:00 查持仓+主力；selection_guard 校验
